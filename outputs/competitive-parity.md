## Differentiation

MergeProof's primary novelty is a **merge evidence ledger**, not another ungrounded review chatbot:

1. Every model citation must resolve to a fetched PR source and the exact PR head SHA or working-tree digest.
2. Local repository retrieval is rejected when its index is stale or belongs to another GitHub remote.
3. Missing evidence produces an explicit abstention state instead of a confident approval.
4. Deterministic security findings cannot be removed by model output and are included in every publication path.
5. Review memory is local, bounded, redacted, and inspectable rather than silently retained by a hosted service.
6. Fixes are tested in an ephemeral Git worktree before they can be reported as verified; the developer checkout is not mutated.
7. Every completed analysis emits a reproducible SHA-256 attestation over the decision, evidence rows, security findings, and PR head SHA or working-tree digest.
8. Independent model disagreement is surfaced as a first-class merge risk; consensus never hides minority evidence.
9. The walkthrough is generated from fetched change evidence rather than invented runtime behavior; its Mermaid diagram is explicitly labeled as a change flow, not a production execution trace.
10. Named finishing touches are configuration-reviewed and path-scoped; a recipe cannot silently expand its write surface beyond its configured scope.
11. Outcome feedback closes the loop between a merge decision and what happened after merge, allowing teams to measure ready-decision calibration instead of optimizing only for review volume.
12. Saved analyses can be independently attestation-verified after transport or publication, making tampering observable instead of trusting the displayed decision.
13. A local natural-language implementation request is treated as a reproducible, evidence-bounded patch job: clean HEAD, bounded retrieval, ephemeral worktree, verification, optional re-review, and stale-checkout refusal before apply.
14. Free-form planning uses the same ledger boundary as review: a plan is not allowed to retain citations from a different checkout or commit, so planning and implementation start from the same verifiable evidence surface.
15. Natural-language pre-merge checks are evaluated as normal evidence rows, so teams get a measurable abstention when a custom rule cannot be proven rather than an opaque pass/fail toggle.
16. Follow-up issues are duplicate-safe and enriched with related issue links, available smart labels, and optional configured assignees without inventing labels that do not exist in the repository.
17. Review capsules make a completed decision independently auditable after the provider or model is gone: the context, citations, exact head SHA, analysis attestation, and bundle digest travel together and can be checked offline.
18. Existing CI security and quality tools can join the evidence ledger through bounded SARIF artifacts; MergeProof does not need shell access to reproduce or trust a tool result.
19. Central policy inheritance keeps organization defaults reviewable as code while preserving repository-local overrides and the same bounded custom-check contract.
20. The maintained PR summary is deliberately marker-scoped: it gives teams a refreshable review narrative without allowing an automation run to overwrite the human-authored PR description.
21. Fleet answers and plans are only comparable when every agent observes the same repository head; context drift becomes an explicit error instead of a hidden multi-agent inconsistency.
22. LSP diagnostics can enter the same evidence ledger as SARIF without granting MergeProof permission to spawn arbitrary language-server processes.
23. Root-level `plugin.json` packaging exposes the review agent through the standard agents/commands/skills distribution shape.
24. ACP exposes a standard editor/IDE transport while preserving MergeProof's safe read-only mode boundary; a client can integrate without depending on the CLI's terminal UI.
25. Autopilot is a bounded correction loop, not an unbounded `yolo` flag: each attempt is sandboxed, verified, re-reviewed, and recorded before a converged patch can be applied.
26. Permission decisions are repository policy, not hidden interactive state; action and path restrictions are inspectable and enforced before mutation or publication.
27. The agent review stream makes evidence and attestation consumable by external coding agents without downgrading to unaudited plain text.
28. Research is a source-pack operation with an explicit opt-in network boundary, so a missing credential becomes a visible limitation rather than an untracked web lookup.
29. Finding disposition is separated from finding evidence: ignoring a result changes review state without deleting the original cited observation, preserving a durable human decision trail.
30. Repository initialization is idempotent and policy-aware, giving a new team a safe local control plane without silently overwriting existing instructions or credentials.
31. `benchmark` turns the evidence ledger into a measurable quality loop: teams can compare review reliability and calibration offline before trusting automation on production repositories.
32. Remote steering is a signed read-only protocol with a first-class client, not an exposed unauthenticated HTTP endpoint; mutation remains local and explicitly permission-gated.
33. The repository is self-describing: the same checkout can report its installed agent surfaces and search its decision timeline without a hosted control plane.
34. Background execution is a durable local state machine rather than a detached fire-and-forget process: task identity, action allowlist, lifecycle timestamps, exit status, bounded logs, and cancellation survive the terminal that started the work.
35. Session compaction is explicit and reversible: older turns move to an inspectable archive, the active transcript retains a recent context window, and a digest-bearing checkpoint makes the context boundary auditable.
36. Review knowledge has a human approval boundary: Slack users and explicitly authorized automation can propose a learning, but no pending proposal can influence a merge decision until a deliberate approval is recorded.
37. Provider approval is a proof-carrying operation: the publication boundary revalidates head freshness, attestation integrity, and review-thread state, so a stale, tampered, or unresolved `ready` artifact cannot become a GitHub approval.
38. Overrides are not hidden switches: a waiver is scoped to one configured check and one exact head, records who/why, travels in the attestation, and expires automatically when the PR head changes.
39. LSP is treated as an evidence source with a process boundary: configuration is inspectable and executable availability is testable, but diagnostics cannot enter a decision unless supplied as a bounded, repository-local artifact.
40. Completion is deliberately non-mutating and freshness-aware: the editor receives an insertion plus the exact source digest used to generate it, so a suggestion can be rejected when the buffer changed before insertion.
41. Prompt replay is opt-in and bounded: source-bearing prompts are never silently persisted, and each saved prompt carries a digest and explicit action/model metadata for auditability.
42. Statistics are derived from the same bounded audit, finding, and outcome ledgers rather than a hidden hosted telemetry stream, so teams can inspect how every number was produced.
43. CodeRabbit adoption is explicit and reversible: a bounded importer maps review intent into MergeProof policy, reports unsupported hosted settings, and refuses to overwrite an existing policy without an explicit force flag.
44. Background task results are durable artifacts separate from logs: status, exit code, completion time, output size, and output digest remain machine-readable after the terminal exits.
45. Configuration migration preserves behavior rather than just names: imported include/exclude globs constrain the evidence context and multiline path guidance survives into the model instruction contract.
46. CodeRabbit finishing touches are portable: bounded custom recipes migrate into the same sandboxed, path-scoped, verification-aware recipe engine instead of becoming inert configuration.
47. Automatic review policy is enforced at the webhook boundary: disabled, draft, incremental, title, author, base-branch, label, and commit-pause rules stop work before a model or publication side effect is attempted.
48. The same bounded autopilot correction loop is available from CLI, desktop, and VS Code: isolated attempts can converge only through verification and evidence re-review, and checkout mutation remains an explicit final action.
49. Shadow mode is a rollout safety valve: teams can observe the exact evidence gate and attestation on live pull requests without allowing an automated review request to block a merge, then switch one policy field to enforcement.
50. PR lifecycle actions share the same guardrails: creating a PR requires a clean branch and optional verification, fixes run in an isolated worktree, and autopilot can only apply a converged verified patch.
51. CodeRabbit-style warning checks remain visible without masquerading as merge-blocking errors; the publication layer distinguishes non-blocking uncertainty from evidence failures.
52. Post-merge action output is durable and digest-addressed, while webhook execution is disabled unless the operator explicitly enables it; a model report cannot execute arbitrary repository commands.
53. Delegated implementation is proof-carrying: the handoff intent, initial repository head, verification contract, correction attempts, patch digest, and apply outcome remain linked in a durable artifact instead of disappearing into a detached agent process.
54. Remote delegation is signed and capability-limited: the server accepts only a bounded request, a known verification command, and explicit lifecycle operations; a remote caller cannot turn the endpoint into arbitrary shell execution.
55. The hosted delegation workflow preserves the same evidence gate across execution locations: local desktop, terminal, signed remote server, and GitHub Actions all emit the same proof-bearing result before a PR can be opened.

## Remaining Deliberate Gaps

MergeProof is not a literal replacement for proprietary hosted surfaces. It does not reproduce Copilot's GitHub-managed Mission Control UI, entitlement/policy plane, or hosted model fleet; its GitHub Actions workflow is an explicit deployable alternative rather than GitHub's native cloud-agent runtime. It also does not reproduce CodeRabbit's hosted knowledge administration, managed billing/tenant controls, backend reachability, or hosted dashboard service. Its equivalents are explicit and inspectable: bounded evidence, agent event streams, ACP interoperability, signed read-only and delegated session steering, resumable/exportable sessions, durable allowlisted background tasks, proof-carrying delegated implementation, bounded autopilot correction, permission policy, opt-in research, parallel fleet/quorum runs, verified ephemeral worktrees, language-server/SARIF diagnostics, local outcomes, non-mutating completion, and conservative mutation boundaries.
